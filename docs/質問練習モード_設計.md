# 質問練習モード 設計ドキュメント

## 1. 目的
パターンプラクティスに「答えを練習／質問を練習」の切替を追加する。質問練習モードでは、お手本の質問文（例: `Do you like dogs?`）をそのまま声に出して再現する練習を行い、しっかり採点する。

ニュアンス: 答えを覚えるか／質問文を覚えるかの違い。
- 答えモード: 質問「Do you like dogs?」に対し「Yes I do」等を練習
- 質問モード: 「Do you like dogs?」という質問文自体を覚えて練習

## 2. 確定仕様
- 「答えを練習／質問を練習」トグルを追加
- 質問練習 ＝ 質問音声を聞く → 生徒が質問文そのものを録音
- 1巡目・2巡目（フォローアップ FQ）とも質問練習の対象
- 対応範囲: HTML版(`public/practice-v2.html`)・React版(`src/components/PracticeMode.tsx`)の2系統
- 採点は既存 `score-answer`(LLM) を流用。質問モードは「お手本＝質問文／厳め（語順・単語に忠実）」で採点。答えモードの採点は従来どおり据え置き
- 文字起こしは既存 `/api/transcribe`(Whisper) を流用
- 宿題(homework)は今回スコープ外 ＝ 答えモード固定

## 3. データ構造の対応
| 概念 | React (`Pattern`) | HTML (`card` / `state`) |
|---|---|---|
| 質問テキスト(1巡目) | `fpp_question` | `card.trigger` |
| 質問音声(1巡目) | `?type=fpp_question` (`has_fpp_question_audio`) | `card.triggerAudio` |
| 答えテキスト(1巡目) | `spp` | `conclusion` |
| 答え音声(1巡目) | `?type=spp` (`has_spp_audio`) | `conclusionAudio` |
| 質問テキスト(2巡目 FQ) | `followup_question` | `followup` |
| 答えテキスト(2巡目 FA) | `followup_answer` | `conclusion2Examples[0]` |

質問・答えは元々分離保持されている。録音対象の照合先を切替えるだけで実現可能（データ追加は不要）。

## 4. 状態・設定
- 新変数 `practiceTarget`（値 `'answer' | 'question'`）。`inputMode`(voice/text)・`turn2Mode`(listen/speak) と直交する独立軸
- localStorage キー `pp-practiceTarget`（2系統で共有）
- デフォルト値 `'answer'`（後方互換 ＝ 従来挙動）

## 5. UI
- 設定パネルの「入力モード」セクション付近に「練習する対象」セクションを追加（2系統とも）
- トグル文言: 「答えを練習」「質問を練習」
- ヒント文言: 「お手本の質問文をそのまま声に出します」

## 6. フロー（答えモード対比）
| 項目 | 答えモード（従来） | 質問練習モード |
|---|---|---|
| 質問音声再生・バブル表示 | あり | 同じ（変更なし） |
| 録音対象 | 答え | 質問文そのもの |
| 採点お手本(`exampleAnswer`) | `spp` / `conclusion` | `fpp_question` / `trigger`（2巡目は `followup_question` / `followup`） |
| 採点モード | 従来（甘め・意味重視） | `scoreMode='question'`（厳め・原文再現） |
| 結果照合表示 | 答え文と対比 | 質問文と対比 |

音声再生・バブル表示は完全に共通。変わるのは「録音後に何と照合するか」のみ。

## 7. 採点API（`src/app/api/score-answer/route.ts`）
- リクエストに `scoreMode: 'answer' | 'question'` を追加（省略時 `'answer'` ＝ 後方互換）
- 答えモードのプロンプトは一切変更しない
- 質問モード用プロンプト方針:
  - お手本質問文との再現度を厳め評価（語順・単語・主要文法）
  - 句読点・大文字小文字・`gonna`/`going to` 等の口語表記揺れは減点しない
  - 意味が同じでも単語・語順が違えば減点（言い換えOKの答えモードと異なり、原文再現が目的）
- レスポンス `{ level, chunkUsed }` の形は維持（呼び出し側互換）

### 質問モード用プロンプト案
```
お手本の質問文: "${exampleAnswer}"
生徒の発話(文字起こし): "${userAnswer}"

生徒はお手本の質問文をそのまま声に出して再現する練習をしています。
どれだけ忠実に再現できたかを厳しめに評価してください。

評価基準:
- perfect: 語順・単語がお手本とほぼ完全一致（句読点・大文字小文字・gonna/going to等の口語表記揺れは無視）
- great: 1語程度の軽微な違い（冠詞・時制の些細なズレ）はあるが質問として同一
- good: 主要語は再現できているが語順や単語に複数の違いがある
- almost: 一部しか再現できていない／意味は近いが文として崩れている
- retry: 全く違う／無音

注意: 意味が同じでも単語・語順が違えば great 以下にする（原文再現が目的）。
JSON形式のみ出力: {"level":"perfect","chunkUsed":true}
```

## 9. 宿題(homework)モードとの切り分け
- 今回は通常のパターン練習のみ対象。宿題は答えモード固定（従来通り）
- 実装: 採点呼び出し直前で `const effectiveTarget = isHomework ? 'answer' : practiceTarget;` とし、宿題時は強制的に答えモード
  - React は `isHomework` prop で判定
  - HTML は URL パラメータ `homework=1` で判定
- この1箇所のガードでフラット型・チャンク型いずれも追加改修不要
- 将来「質問練習宿題」を出す場合は、宿題データ or URL パラメータに練習対象を持たせる別設計が必要（今回は持ち込まない）

## 10. 決定事項（壁打ち反映の推奨デフォルト）
- (A) 質問モードでは `turn2Mode=listen` でも2巡目の質問を録音対象にする（仕様「2巡目も練習」を厳守）
- (B) React の2巡目も質問モード時は API 採点に格上げ（しっかり採点の要望に合致）。答えモードの React 2巡目は従来の自己評価据え置き
- (C) `practiceTarget` は localStorage で永続（宿題はガードで答え固定）
- (D) 質問音声が無い問題は原文テキストを早めに表示して練習継続

## 11. エッジケース・懸念
- 質問音声が無い問題: 既存も音声無しでバブル表示のみ→フォールバック待ち。質問モードでもテキスト表示で継続可。(D)の原文先出しで補助
- Whisper の口語正規化（gonna 等）: 採点プロンプトで「表記揺れは無視」と明示して吸収
- ローカルフォールバック（API失敗時）: 答え用ロジックのため、質問モードでは質問文を渡す対応が必要 → §12-2 で要確認
- 2系統の挙動差: HTML は元々2巡目を API 採点、React は自己評価のみ。(B)で React も質問モードのみ採点に揃える
- 後方互換: `pp-practiceTarget` 未設定時 `'answer'`、`scoreMode` 省略時 API 従来動作 ＝ 既存ユーザー無影響
