01 ペラトレとは

MasaEnglishとは別の、独立したオンライン英会話サービス

先生と受講生がマンツーマンでレッスンを行うオンライン英会話。受講生は毎週固定 or 都度予約でレッスンを受ける。

レッスンの復習教材として「パターンプラクティス」がある。「これを言われたら → これを返す」をシチュエーション付きで繰り返し練習するアプリ。

足りていないこと

今の教材は全員同じ内容。先生がレッスンで「この受講生にはこのフレーズが必要」と気づいても、その受講生だけに教材を追加する仕組みがない。

03 作りたいもの

パターンプラクティスの教材を、先生と受講生が連携しながらカスタマイズできる仕組み

デフォルトの教材を用意しつつ、先生がレッスンで扱った内容をそのまま教材化し、受講生ごとにカスタマイズされた復習教材を自動で届ける。

具体的なイメージ

たとえば、レッスンで挨拶の練習をしたとき：

会話の例

How's it going?

I'm pretty good. And you?

Yeah, I'm pretty good too.

こういった会話を、パターンプラクティスと同じUIで学習できるようにしたい。先生がラフに送った内容が、自動で教材として整形される。

3つの柱

  • 閲覧/編集モードの切り替え — 受講生は閲覧モードで学習、先生は編集モードで内容を更新（パスワード認証付き）
  • 受講生ごとの個別サイト — 受講生がGoogleログインすると、自分専用の教材が表示される。個別の学習進捗や内容を管理
  • レッスン内容のシームレスな転送 — 先生がラフに内容を送る → AIが自動で教材フォーマットに整形・音声生成 → 受講生のサイトに自動反映

全体の流れ

先生がレッスン後にフォームでメモを送信（ラフでOK）
↓
AIが自動で教材フォーマットに整形（GPT-4o-mini）
↓
ElevenLabsで音声を自動生成
↓
DBに保存、受講生に紐付け
↓
受講生がログインすると自分専用の教材ができている
最重要ポイント：先生の手間を極限まで減らす 画面共有しながらフィールドを一つずつ埋めるのは非常に使いづらい。テキストエリア1つにラフに会話を書いて送るだけ。AIがtrigger/回答/followupの分解、situation生成、カテゴリ分類、音声生成まで全部やる。先生はフォーマットを意識する必要すらない。面倒だと使わなくなる。
04 やってほしいこと

先生用フォーム + AI整形 + 音声自動生成

最優先

これがないと何も始まらない。先生がレッスン中に画面共有しながらサッと書けるフォーム。送ったら裏でAIが教材に整形して音声も自動生成する。

フォームの入力イメージ

フィールドを分けない。テキストエリア1つにラフに書くだけ。AIが中身を解析して教材に分解する。

ドロップダウンで選ぶ
How's it going? I'm pretty good. And you? Yeah, I'm pretty good too. 週末の予定聞かれた場面で使った

先生はこれだけ。会話とメモをテキストエリアにラフに書いて送信するだけ。フィールドを一つずつ埋める必要はない。画面共有しながらでもサッと書ける。

送信後にAIが自動でやること
  • 会話の分解（GPT-4o-mini）— ラフなテキストからtrigger（相手のセリフ）・回答・followupを自動で判別・分解
  • 教材整形 — situation（状況説明）を自動生成、カテゴリ自動分類。雑な入力でもちゃんとした教材フォーマットに落とし込む
  • 音声生成（ElevenLabs）— 分解した各セリフの音声を自動生成
  • DB保存 — patternsテーブルに教材データ、audio_filesに音声、assignmentsで受講生に紐付け
先生の認証

先生用ページにはアクセス制限が必要。パスワード認証が.env.localに設定済み（TEACHER_PASSWORD）

受講生Googleログイン + 個別教材表示

優先度2

受講生がGoogleアカウントでログインすると、自分専用の教材が表示される。

  • Google OAuth認証を実装
  • 教材 = 全員共通のベースフレーズ + 先生が追加した個別フレーズ
  • 受講生テーブル: Google ID、名前、メールアドレス、担当先生
  • ログインしていない場合はベースフレーズのみ表示

ベースフレーズのDBインポート

優先度3

現在practice-v2.htmlに埋め込まれているフレーズデータをDBにインポート。seed-patternpractice.mjsがベース。

先生向け使い方動画の作成

優先度4

フォームが完成してから、先生向けに「このツールをどう使うか」を説明する動画を作成する。先生がフォームを見てすぐ使えるように、操作手順をわかりやすく伝える。

06 開発環境セットアップ

Claude Codeで開発する前提。以下をやればローカルで動く。

  1. リポジトリをクローン
    git clone https://github.com/naomasafukumura/peratore-dashboard.git
  2. 依存関係インストール
    npm install
  3. 環境変数を設定
    .env.local.example をコピーして .env.local を作成。DATABASE_URL等の値はMasaから共有
  4. DBセットアップ
    node db-setup.mjs でテーブルを作成
  5. 初期データ投入
    node seed-patternpractice.mjs でベースフレーズを投入
  6. 開発サーバー起動
    npm run dev → http://localhost:3000

DBスキーマ（既存テーブル）

-- カテゴリ（例: 返答型 > 返答：未来） categories: id, type, name, sort_order -- チャンク（例: I'm gonna ~） chunks: id, category_id, chunk_number, title_en, title_jp, sort_order -- パターン（実際の練習問題） patterns: id, chunk_id, set_number, situation, fpp_question(trigger), spp(模範回答), followup_question, followup_answer, character(ボイス判定用), ... -- 音声ファイル audio_files: id, pattern_id, audio_type, voice_id, audio_data(BYTEA), ... -- 受講生割り当て assignments: student, chunk_id

詳細は db-setup.mjs を参照
